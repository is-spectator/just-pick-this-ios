# App Foundation V1

本轮补齐 iOS App 的基础使用框架，不改皮皮 Agent 和后端协议。

> 2026-06 产品化更新：早期“三 Tab”壳已经被 Chat-first App Shell 取代。App 启动后直接进入皮皮聊天；来一句、我的求一个、我的回答、收藏、奖励、消息中心和账号入口统一收进左侧 Drawer。

## 新增

- 单一 Chat 主界面：顶部菜单 / 皮皮 / 新对话，底部 Composer。
- 左侧 Drawer：搜索、新对话、来一句、我的求一个、我的回答、收藏、奖励、消息中心、置顶、最近和账号入口。
- Drawer 搜索进入实时结果模式：按历史会话、求一个、收藏分区展示匹配内容，无结果时给短空状态。
- Drawer 产品页：我的求一个、我的回答、收藏、奖励、消息中心已拆成独立页面，不再统一塞进“我的”页。
- 主界面菜单按钮会在有亮灯消息时显示红点，消息中心打开后清除本机未读提示。
- Chat 顶部新增决策地点条：用户可以明确选择“使用当前定位”或手动输入城市/区域/地标，当前地点会显示在主聊天页。
- iOS 只在用户点击“使用当前定位”时请求系统定位授权；发送消息时会把选择的地点通过 `client_context.decision_location` 传给 `/v1/chat/turn`。
- 后端 InputGate 会在用户只说“我想吃川菜”这类缺地点表达时合并 `decision_location`，避免已经选了地点还反复追问。
- Chat Composer 增加显式键盘焦点管理：发送、打开菜单/地点选择、点击或拖动消息区都会收起键盘，减少真机上键盘挡住内容的情况。
- 推荐卡 `…` 菜单接入产品动作：收藏会进入 Drawer 收藏页并支持取消；“不合适，换一个”“信息有误”“求一个”会调用后端卡片反馈接口，写入效果评测事件。
- 我的求一个页新增状态概览：草稿、收集中、已有结果、已完成/已关闭数量在 Drawer 入口内可直接查看。
- 我的回答页新增答主状态桶和最近提交记录：待采纳、已采纳、未采用与奖励状态关联，用户写过的一句会在本地立即可见。
- 奖励页新增明细：读取 `/v1/rewards/me` 的奖励流水，并把刚提交但还未回流的本地来一句显示为待确认奖励。
- 我的求一个新增详情页：从记录进入可查看求助内容、人类回答和皮皮最终推荐，不暴露 Agent 技术日志。
- 我的页面：账号、积分、回答次数、历史、求一个、消息预览和版本，作为 Drawer 里的账号/设置入口承载。
- 账号与设置集中在“我的”页：支持邮箱登录入口、隐私与数据说明、退出登录和删除账号，不把设置动作散落在聊天页。
- 删除账号走后端 `DELETE /v1/auth/me` 软删除：撤销当前会话、清空邮箱登录信息，并把本机 token 清除。
- Profile API 聚合：奖励、答主质量、亮灯消息；消息中心复用 light-events 列表。
- API Base URL 改为可由 Info.plist 配置。
