# App Foundation V1

本轮补齐 iOS App 的基础使用框架，不改皮皮 Agent 和后端协议。

> 2026-06 产品化更新：早期“三 Tab”壳已经被 Chat-first App Shell 取代。App 启动后直接进入皮皮聊天；来一句、我的求一个、我的回答、收藏、奖励、消息中心和账号入口统一收进左侧 Drawer。

## 新增

- 单一 Chat 主界面：顶部菜单 / 皮皮 / 新对话，底部 Composer。
- 顶部「新对话」改为明确的图标+文字胶囊入口，避免只用 `square.and.pencil` 图标让用户猜用途，同时保持顶部只露出菜单和新对话两个操作。
- 左侧 Drawer：搜索、新对话、来一句、我的求一个、我的回答、收藏、奖励、消息中心、置顶、最近和账号入口。
- Drawer 搜索进入实时结果模式：按历史会话、求一个、收藏分区展示匹配内容，输入后可一键清空，无结果时给短空状态。
- Drawer 会话管理补齐本机持久化：置顶、重命名和删除会保存到本机，并在 Drawer 内给出短反馈；搜索结果里也可以长按直接管理。
- Drawer 最近会话按真实创建时间分组：今天、7 天内、更早不再按列表数量硬切；本地新会话和后端 `created_at` 历史都能进入对应分组。
- Drawer 产品页：我的求一个、我的回答、收藏、奖励、消息中心已拆成独立页面，不再统一塞进“我的”页。
- 主界面菜单按钮会在有亮灯消息时显示红点，消息中心打开后清除本机未读提示。
- Chat 顶部新增决策地点条：用户可以明确选择“使用当前定位”或手动输入城市/区域/地标，当前地点会显示在主聊天页。
- 决策地点选择补齐“搜索或输入”体验：地点 sheet 会展示最近使用地点和常用地点建议，输入时实时过滤，点选后直接作为当前决策地点。
- 决策地点输入态补齐：手动地点支持一键清空，未输入时保存按钮禁用，避免空地点误保存。
- iOS 只在用户点击“使用当前定位”时请求系统定位授权；发送消息时会把选择的地点通过 `client_context.decision_location` 传给 `/v1/chat/turn`。
- 后端 InputGate 会在用户只说“我想吃川菜”这类缺地点表达时合并 `decision_location`，避免已经选了地点还反复追问。
- Chat Composer 增加显式键盘焦点管理：发送、打开菜单/地点选择、点击或拖动消息区都会收起键盘，减少真机上键盘挡住内容的情况。
- Chat 错误状态支持重试：服务没连上时会把原话回填到输入框，消息内提供重试入口，用户也可以先改一句再发。
- 顶部返回按钮触控区域统一到 44pt，和菜单、新对话、发送等主操作保持同一点击规范。
- 推荐卡 `…` 菜单接入产品动作：收藏会进入 Drawer 收藏页并支持取消；取消收藏会给出短 notice 和轻触感，不再静默消失；分享会拉起系统分享面板；“不合适，换一个”“信息有误”“求一个”会调用后端卡片反馈接口，写入效果评测事件；聊天流推荐卡里的“求一个”也会同步写入 `ask-human` 反馈。
- 结果页推荐卡菜单补齐可见反馈：收藏和信息有误会在卡片上方显示短 notice，不再只靠状态静默变化。
- 卡片长文本保护：推荐卡和求一个标题限制在 2–3 行内，长正文默认折叠并支持展开，避免卡片被超长标题或回答撑爆。
- 推荐卡图片加载改为轻量骨架态：图片区域保持稳定高度，不再用局部转圈打断阅读。
- 来一句 Deck 加载态改为卡片骨架：保持卡片尺寸和层级稳定，不再在卡中央显示 spinner。
- 来一句 Deck 输入框接入父层焦点控制：点击卡片区域或拖动列表会收起键盘，发送一句后也会主动收起。
- 来一句成功反馈补齐自动切卡说明：提交后明确告诉用户已经切到下一张，若队列为空也给出短提示。
- 来一句 Deck 当前卡右上角接入 `…` 菜单：提供举报和屏蔽入口；举报/屏蔽会调用 `/v1/help-cards/{id}/skip` 写入后端信号，再移出当前卡并继续下一张，避免把低频操作做成常驻按钮。
- 我的页里的历史、选择和消息空态统一为产品页 inline empty panel，不再只显示一行弱提示文字。
- 产品页刷新入口统一为 44pt toolbar control：我的、我的回答、奖励和消息中心的刷新/加载态使用同一触控与 VoiceOver 规则。
- 我的求一个页新增状态概览和专用记录行：草稿、收集中、已有结果、已完成/已关闭数量在 Drawer 入口内可直接查看，列表里也会显示对应状态和下一步说明。
- 我的回答页新增答主状态桶和最近提交记录：待采纳、已采纳、未采用与奖励状态关联，用户写过的一句会在本地立即可见。
- 我的回答页修正非交互总览卡：只读状态卡不显示右箭头，避免暗示可点击。
- 我的回答页最近提交记录可进入原求助详情：用户能回看自己回答的问题、人类回答和最终结果。
- 我的回答页可回答预览支持直达来一句 Deck：点某张预览会把它设为当前卡，再进入一屏一张的回答体验。
- 采纳来一句最终结果接入后端：`采纳这句` 会调用 `/v1/help-cards/{id}/accept-final` 写入最终推荐采纳事件；网络失败时保留本地完成兜底，不打断用户路径。
- 奖励页新增明细：读取 `/v1/rewards/me` 的奖励流水，并把刚提交但还未回流的本地来一句显示为待确认奖励。
- 奖励页明细支持回溯来源：能定位到求助卡的奖励行可进入原求助详情，查看问题、人类回答和最终结果。
- 我的求一个新增详情页：从记录进入可查看求助内容、人类回答和皮皮最终推荐，不暴露 Agent 技术日志。
- 下拉刷新补齐：Drawer 下拉会刷新当前求助状态、来一句队列和消息红点；来一句 Deck 和消息中心支持系统下拉刷新。
- 求助详情支持下拉刷新：查看人类回答和皮皮最终推荐时，不必只依赖右上角刷新按钮。
- Drawer 历史加载补齐骨架态：历史为空且正在刷新时展示稳定的历史行骨架，避免刷新过程中内容区域忽然跳动或误读为空。
- 消息中心初始加载补齐骨架态：空数据加载中不先显示“没有消息”，避免把加载状态误读成空状态。
- 消息中心补齐点击流转：前端保留后端 light event 的 `help_card_id/card_id`，消息行可点击；有求助关联时进入求助详情，有奖励或泛消息时进入对应产品页。
- Light / Dark Mode 基础适配：颜色 token 改为系统语义色，并拆分主操作按钮的背景/前景 token，保证卡片、输入框和主按钮在暗色模式下仍可读。
- 图片失败自动降级：推荐卡图片加载失败或遇到未知图片状态时直接退化成无图文本卡，不显示破图图标或假占位。
- Drawer 转场动画精修：侧边抽屉改为跟手拖拽进度驱动，聊天层、遮罩和抽屉位移同步过渡，松手后使用原生弹簧收尾。
- 推荐卡入场动画补齐：聊天流推荐卡和结果页推荐卡都使用轻淡入上移，不做夸张弹跳。
- 轻触感反馈补齐：Composer 发送、推荐采纳、求一个发布、收藏、分享、反馈、来一句提交和来一句滑卡统一使用轻量反馈；校验失败使用 warning feedback，不做重反馈打扰。
- 无障碍提示补齐：顶部入口、Drawer 行、推荐卡主操作、来一句 Deck 和 Deck 空状态按钮增加 VoiceOver hint，说明动作会打开哪里或产生什么结果。
- 产品页列表语义补齐：收藏、回答记录、奖励明细、空状态和求助详情摘要会被 VoiceOver 合并成可理解的一句话。
- 消息与地点入口补齐 VoiceOver 语义：顶部菜单会读出“有新消息”，Drawer 消息中心会读出未读数量，决策地点条和地点建议会说明会修改当前决策地点。
- 我的页面：账号、积分、回答次数、历史、求一个、消息预览和版本，作为 Drawer 里的账号/设置入口承载。
- 账号与设置集中在“我的”页：Drawer 底部账号入口未登录时进入邮箱登录，已登录时进入我的/设置；支持隐私与数据说明、退出登录和删除账号，不把设置动作散落在聊天页。
- 删除账号走后端 `DELETE /v1/auth/me` 软删除：撤销当前会话、清空邮箱登录信息，并把本机 token 清除。
- Profile API 聚合：奖励、答主质量、亮灯消息；消息中心复用 light-events 列表。
- API Base URL 改为可由 Info.plist 配置。
