# UI Audit 2026-06-16

## Scope

本轮只审计和优化 iOS 原生 SwiftUI 界面。不改后端、不改 Agent 架构、不改 API 协议、不接真实 LLM。

## 当前主要页面和组件清单

- `InputScreen`：首页聊天入口，承载用户输入、推荐卡、求一个卡和历史 sheet。
- `ChatRecommendationCard`：聊天流里的推荐卡，目前以横向缩略图 + 信息摘要呈现。
- `ChatHelpCard`：聊天流里的求一个草稿卡。
- `AnswerScreen`：来一句入口页面，承载卡片 Deck、底部输入框和 toast。
- `HelpDeckStack`：来一句卡片堆叠与左右滑动容器。
- `HelpDeckCard`：来一句单张求一个卡。
- `BottomComposer`：全局底部输入框。
- `TopBar`：顶部导航，包含历史、新对话、来一句和账号入口。

## 当前推荐卡问题

- `ChatRecommendationCard` 使用 104x104 缩略图横排，视觉更像列表项，不像“皮皮拍板”的主卡。
- 标题字号和层级偏小，主选择不够第一视觉。
- 同时渲染 `subtitle`、`reason`、`bullets` 和操作按钮，信息密度偏高，违背“一个选择 + 一个理由”。
- 无图时有 fallback thumbnail，占位感强，容易显得像未加载完成。
- `visibleBullets` 仍会暴露多个补充理由，不利于单一决策体验。

## 当前求一个 / 来一句页面问题

- `AnswerScreen` 顶部缺少页面内语义标题，只有通用返回导航标题。
- `HelpDeckCard` 标题和 context 居中，像低保真展示稿；真实产品卡片更适合左对齐层级。
- 下一张卡露出存在，但可读性和 deck 感还能增强。
- 输入框 placeholder 固定为“来一句,帮 TA 少纠结”，没有结合当前卡内容。
- Loading/empty 文案仍出现“求助卡”等偏工程词。

## 当前导航和文案问题

- 首页顶部同时出现历史、账号、新对话、来一句入口，视觉略拥挤。
- “历史 session”是工程词，应改为“历史”。
- 来一句入口使用 ellipsis 不够明确，应换成更语义化图标。
- 账号入口暂时不应该占主导航视觉中心；标题应更稳定地显示“皮皮”。

## 建议修改文件列表

- `NativeApp/Screens.swift`
  - 重做 `ChatRecommendationCard`。
  - 优化 `AnswerScreen`、`HelpDeckStack`、`HelpDeckCard`、`HelpDeckLoadingCard`、`EmptyAnswerQueueCard`。
  - 增补 previews。
- `NativeApp/Components.swift`
  - 简化 `TopBar` 图标和文案。
  - 强化 `BottomComposer` 点击区域和视觉一致性。
  - 微调 `ToastView` 位置和样式。
- `NativeApp/AppTheme.swift`
  - 可添加少量 surface/shadow token，但不大改色彩体系。

## 边界说明

- 不修改 `backend/`。
- 不修改 API contract。
- 不修改 Agent runtime、reasoner、工具调用或数据模型。
- 不新增第三方 UI 框架。
